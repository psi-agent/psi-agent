## Context

测试文件中存在多处非顶级 import，需要移至顶级以符合 PEP 8 和项目编码规范。

## Goals / Non-Goals

**Goals:**
- 将所有测试文件中的非顶级 import 移至文件顶级
- 保持 import 顺序符合规范（stdlib → third-party → local）
- 确保所有测试仍然通过

**Non-Goals:**
- 不修改生产代码
- 不改变测试逻辑

## Decisions

### 决策 1：统一顶级导入

**选择**：将所有测试文件中的 import 移至顶级。

**理由**：
- 代码一致性：生产代码已统一顶级导入，测试代码也应一致
- 可维护性：import 集中在文件顶部，更易管理
- 依赖清晰：一眼就能看出文件依赖哪些模块

## Risks / Trade-offs

**风险 1**：模块加载失败导致整个测试文件无法运行。

- **影响**：如果某个依赖模块有问题，整个测试文件会加载失败。
- **缓解**：这是正常行为，说明依赖有问题需要修复，而不是隐藏问题。
