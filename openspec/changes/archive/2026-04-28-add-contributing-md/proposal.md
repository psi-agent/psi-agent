## Why

The project needs a CONTRIBUTING.md to guide contributors on how to participate. Given the project's unique context where code quality and consistency are enforced through Claude Code and CLAUDE.md, contributors should use Claude Code for any PR implementation rather than writing code manually.

## What Changes

- Add CONTRIBUTING.md with contribution guidelines
- Clarify that issues and PRs are welcome
- Specify that PR code must be written using Claude Code (human-written PRs are not accepted)
- Reference CLAUDE.md as the source of truth for coding standards

## Capabilities

### New Capabilities

None - this is a documentation-only change.

### Modified Capabilities

None - no existing specs are affected.

## Impact

- New file: `CONTRIBUTING.md` at repository root
- No code changes
- No API changes
- No dependency changes
