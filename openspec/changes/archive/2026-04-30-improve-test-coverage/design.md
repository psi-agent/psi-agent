## Context

当前测试覆盖率 47%，多个模块覆盖率低于 80%。需要为这些模块添加单元测试以提高整体覆盖率。

测试策略：
- CLI 模块：测试参数解析和命令调用
- API 模块：测试核心逻辑路径
- Server 模块：测试请求处理和错误处理

## Goals / Non-Goals

**Goals:**
- 为覆盖率低于 80% 的模块添加测试
- 优先处理覆盖率最低的模块
- 使用 mock 隔离外部依赖

**Non-Goals:**
- 不修改功能代码
- 不追求 100% 覆盖率（不现实的测试成本）
- 不添加集成测试（仅单元测试）

## Decisions

### Decision 1: 优先处理覆盖率最低的模块

**选择**: 按覆盖率从低到高处理：channel/cli/cli.py (39%) → ai cli 模块 (52-53%) → workspace/api 模块 (63-78%)

**理由**: 优先提升最低覆盖率的模块可以最大化整体覆盖率提升

### Decision 2: 使用 mock 隔离外部依赖

**选择**: 使用 `unittest.mock` 和 `pytest-mock` 隔离 Telegram API、文件系统、网络请求等外部依赖

**理由**: 单元测试应该快速、可靠，不依赖外部服务

## Risks / Trade-offs

- **测试维护成本**: 更多测试意味着更多维护 → 使用清晰的测试命名和结构降低维护成本
- **Mock 与实现不一致**: Mock 可能与实际行为不符 → 关键路径使用集成测试验证
