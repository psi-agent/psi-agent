## Context

psi-agent 使用 uv 管理依赖，pyproject.toml 中使用最低版本约束。当前需要更新到 PyPI 最新版本以保持依赖明确性。

## Version Changes

| 依赖 | 当前约束 | 最新版本 | 备注 |
|-----|---------|---------|------|
| aiohttp | >=3.9.0 | 3.13.5 | 小版本升级 |
| anyio | >=4.0.0 | 4.13.0 | 小版本升级 |
| loguru | >=0.7.0 | 0.7.3 | 小版本升级 |
| tyro | >=0.8.0 | 1.0.13 | **大版本升级** |
| pytest | >=8.0.0 | 9.0.3 | 大版本升级 |
| pytest-asyncio | >=0.23.0 | 1.3.0 | **大版本升级** |
| ruff | >=0.4.0 | 0.15.12 | 小版本升级 |
| ty | >=0.0.0 | 0.0.33 | 小版本升级 |

## Goals / Non-Goals

**Goals:**
- 更新所有依赖版本号到 PyPI 最新版本
- 运行质量检查确保无兼容性问题

**Non-Goals:**
- 不添加新依赖
- 不移除现有依赖

## Decisions

### 版本约束策略

使用精确版本约束（如 `==3.13.5`）还是最低版本（如 `>=3.9.0`）？

**决定**：保持最低版本约束风格，但更新下限版本号。例如：
- `aiohttp>=3.13.5`（而非 `>=3.9.0`）
- 这样允许 uv 在锁文件中使用该版本，同时保持未来兼容性

### 大版本升级处理

tyro 1.0 和 pytest-asyncio 1.3 是大版本升级，需要验证：
- tyro 1.0 API 变化：检查 CLI 接口是否兼容
- pytest-asyncio 1.3 API 变化：检查 asyncio_mode 配置是否仍有效

## Risks / Trade-offs

- **tyro 大版本升级**：可能有 CLI API 变化，需测试
- **pytest-asyncio 大版本升级**：可能有配置变化，需测试
